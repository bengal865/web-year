### The "Plain English" Rule for Your Workspace

Each day has ONE specific job. If you are doing something else, you are straddling and working less efficiently.

* **Monday:** Ship
* **Tuesday:** Build
* **Wednesday:** Write
* **Thursday:** Cut
* **Friday:** Reset

---

### 📅 Monday: Delivery Day

**The Goal: Get files into students' hands.**

- Today is not for coding or writing instructions; it’s for moving files
- Take work finished last week, strip out answers and add to wiki

* *Success looks like:* Every student has the right file at the start of class

### 📅 Tuesday: Problem-Solving Day

**The Goal: Make sure it works.**
- This is your "Deep Work" day
- Build next week's lab from scratch
- Don't worry about how to teach it yet; you make sure code is correct and the solution is solid

* *Success looks like:* A finished, working Python script or web page in your private repo.

### 📅 Wednesday: Teaching Day

**The Goal: Explain the "What."**
- Now that the code works, figure out how to explain it
- Write `TODO` instructions and set up the wiki page
- Build the map students will follow

* *Success looks like:* A list of clear goals that tell the students what their objective is.

### 📅 Thursday: Cutting Day

**The Goal: Make it harder (and shorter).**
- The most Essentialist day
- Review what you wrote on Wednesday and ask: *"Am I helping too much?"* 
- Delete extra hints, shorten long sentences, and enclose answers in solution markers

* *Success looks like:* A lean lab that forces students to actually think.

### 📅 Friday: The Clean-Up Day 

**The Goal: Reset for next week.**
- Review how students did this week
  - If everyone got stuck on the same thing, correct it for next time
- Verify links one last time, close your browser tabs, and walk away for the weekend

* *Success looks like:* A clear head and a ready-to-go plan for Monday morning.
