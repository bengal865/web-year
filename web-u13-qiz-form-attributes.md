## 013 HTML5 Forms
### Quiz: Form Attributes

**1. The "Ghost" Data**
You want to show an example email address inside a text box that disappears when the user clicks it. Which attribute should you use?

* A) `value`
* B) `placeholder`
* C) `id`

**2. The Data Delivery**
You hit "Submit," but the URL only says `index.html?` and the URL is missing all the text you typed in. What attribute did you likely forget to add to your `<input>` tags?

* A) `name`
* B) `required`
* C) `autocomplete`

**3. The Choice of One**
In a list of three radio buttons, you want the middle option to be already filled in when the page loads. Which attribute do you add to that specific input field?

* A) `autofocus`
* B) `required`
* C) `checked`

**4. ID vs. Name**
If you want to use CSS to turn a specific input box **neon green**, which attribute is the best one to use as your selector?

* A) `id`
* B) `name`
* C) `value`

**5. The Wall**
Which attribute allows a user to see the information in a box but prevents them from changing it, while still sending that data when the form is submitted?

* A) `disabled`
* B) `readonly`
* C) `placeholder`

---

### ✅ Answer Key (For Your Eyes Only)

1. **B** (`placeholder`)
2. **A** (`name` — the server needs the name to identify the data)
3. **C** (`checked`)
4. **A** (`id` — IDs are unique and intended for styling/scripts)
5. **B** (`readonly` — `disabled` fields are not sent to the server)
